Part 1: Take-home Assignment
Project: Earth Observation (EO) Data Pipeline & Pulse Map
Objective
Build a mini-application that processes a provided dataset of Earth Observation (EO) anomalies (e.g., surface temperature shifts) and visualizes them on an interactive dashboard. The candidate should demonstrate how they use AI to accelerate development without sacrificing data integrity.
Stack
* Backend: Python (Django or FastAPI preferred).
* Frontend: jQuery, React, Vue, or vanilla JS with a mapping library (Leaflet, Mapbox, or OpenLayers).
* AI: Use of LLMs (ChatGPT, Claude, Copilot, etc.) is encouraged but must be documented.
________________


Task 1: Data Processing & Automation
The Scenario: You are provided with "messy" CSV files containing sensor readings (the sensor records 1 observation per minute, but some observations might be missing).
* Problem: The file names follow a strict but complex naming convention (e.g., SENSOR_A-20231012-99234-TEMP.csv).
* Requirement: Write a Python script to:
   1. Parse the file names to extract the Numeric ID (the second-to-last part) and the Date.
   2. Find all anomalies, where the sensor reading is outside a "scientific" range (e.g., -50 to 60).
   3. Expose this cleaned data via a simple REST API endpoint.
Task 2: The Interactive Visualization
The Scenario: Scientists need to see where these anomalies are happening in real-time.
* Requirement: Build a web page that:
   1. Consumes your API.
   2. Displays an interactive map showing the sensor locations.
   3. Uses color-coding (a heatmap or styled markers) to represent the intensity of the reading.
   4. Includes a dynamic chart (using Chart.js or D3) that shows the trend of readings over time when a sensor is clicked.
Task 3: AI Collaboration Log
The Scenario: We want to see how you work with modern tools.
* Requirement: Provide a short "AI Log" (or commented code) explaining:
   * Which parts of the code were generated by AI?
   * Where did the AI fail or "hallucinate" (e.g., wrong library syntax, inefficient data handling)?
   * How did you prompt the AI to handle the specific geospatial requirements?
Task 4: Architecture & Design
The Scenario: We need to scale this to 100,000 sensors.
* Requirement: Write a brief (2-paragraph) README section explaining how you would adapt this architecture to handle massive Earth Observation datasets. Mention specific strategies for frontend performance and data storage.


Deliverables


  A GitHub repository link.
  A brief README with setup instructions.
  The "AI Collaboration Log"

Part 2: Technical Assessment Questions
1. AI Implementation & Agentic Workflows
* The "Context Window" Strategy: "We are building an agent to analyze a 50-page methodology document and answer questions about a specific web map's data. How do you decide what goes into the prompt context versus using a RAG (Retrieval-Augmented Generation) approach?"
* Prompt Engineering for Data: "How would you structure a prompt for an LLM to reliably transform a messy scientific metadata string into a structured JSON object that matches our internal schema?"
* AI as a Coworker: "Describe your 'Human-in-the-Loop' philosophy. When should an AI agent act autonomously, and when must it stop to ask a developer or scientist for verification?"
2. Geospatial & Scientific Visualization
* Handling Scale: "If we need to render 100,000 active data points on an interactive map without crashing the user's browser, what front-end strategies (e.g., clustering, tiling, WebGL) would you evaluate first?"
* The Projection Trap: "A scientist complains that their data looks 'stretched' or 'off-center' on your web map. How would you debug coordinate reference system (CRS) mismatches between the back-end GeoJSON and the front-end map library."
* Dynamic Charting: "When visualizing a time-series of Earth Observation data (like 20 years of soil moisture levels), how do you handle 'missing' data points in a way that remains scientifically accurate but visually intuitive for the user?"
3. Architecture & Efficiency
* The Design System ROI: "If I give you a budget to build a custom design system for our suite of apps, how would you measure whether that investment actually made your fellow engineers more efficient six months later?"
* Python/Django Integration: "We use Python for heavy data lifting. How do you architect the hand-off between a heavy geospatial processing task in Python and a responsive React or Vue front-end?"
